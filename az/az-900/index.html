Regions and Geographicsâ€‹

Una region es un grupo de multiples datacenters cada datacenter se lo conoce como un availability zone.

En total azure tiene 60 regiones en 140 paises

Por otro lado las geografias son un mercado conformado de 2 o mas regiones que preserva el estado de la informacion y los limites de confianza

Tenemos geografia en:

  • canada
  • usa
  • govierno de usa
  • brazil
  • mexico
tip

Paired regions Azure agrupa las regiones en pares con el fin de que puedan ser fault tolerance y que si una region esta siendo actualizada o en mantenimiento no sufras dichos downtimes. Normalmente estas regiones estan separados 500 km

Fault Domain Es la forma en la que realizamos las agrupaciones del hardware con el fin de evitar single point of failure. Comparten el power supply y el switch

Update Domain Azure necesita realizar actualizaciones o mantenimiento en su hardware es por esta razon que los update domain aseguran que no tengas downtimes

Azure resource groupâ€‹

Un resource group es una forma de manejar tus recursos como una sola entidad de forma que tengas mas control y orden de la infraestructura que se provisiona, es decir, es un contenedor en donde podemos agregar y manejar los recursos necesarios para nuestro caso de uso en una sola unidad. De esta forma podemos manejar las aplicaciones y no manejamos los componentes de las aplicaciones.

Antes de poder provisionar cualquier recurso en la nube necesitamos agregarlo a un resource group

warning

Cuando eliminamos un resource group eliminamos todos los recurso que contiene

Tenemos que tener en cuenta las siguientes reglas para los resource group:

  1. Cada recurso debe ser unico en el resource group
  2. Un recurso solo puede existir en un resource group
  3. Un resource group puede contener recursos que existen en otras regiones
  4. El usuario decide que recursos quiere que pertenezcan a cada grupo dependiendo de que le haga mas sentido para su caso de uso
  5. Podemos agregar y eliminar recursos en cualquier momento
  6. Podemos mover recursos de un resource group a otro sin problemas
  7. Recursos para una aplicacion no es necesario que pertenezcan al mismo resource group pero es recomendable que si lo haga con el fin de que sea mas facil manejar la aplicacion

Azure Resource Manager (ARM)â€‹

Es una forma de tener la infraestructura como codigo, de modo que podamos crear siempre lo mismo a partir de un script. Usamos el lenguaje JSON para crear dichos scripts. Ademas tenemos las siguiente funciones principales

  1. Deployar recursos de las aplicaciones
  2. Organizar recursos
  3. Controlar el accesso a nuestros recursos

Son los ARM templates que la comunidad ya a hecho con el fin de reutilizarlo y no empezar de 0. Tenemos estos templates para cosas basicas, como crear una base de datos de postgres o levantar una vpn con subnets dentro.

Azure compute es un servicio de computacion bajo de manda para ejecutar nuestras aplicaciones basadas en la nube. Provee de recursos como discos duro, memoria, procesadores, sistemas operativos, etc.

  • Azure virtual machines*: el mas sencillo de todos tenemos maquinas virtuales que podemos usar

  • Azure container instances*: Desplegar imagenes de docker sin provisionar servidores

  • Azure kubernetes services: facil de desplegar, manejar y monitorear clusters de kubernetes

  • Azure services fabric: PLataforma para sistemas distribuidos hecha para microservicios

  • Azure functions: Serverless subimos funciones de programacion, son basadas en eventos y solo pagamos el tiempo que la funcion se esta ejecutando

  • Azure batch: Para crear un batch de maquinas virtuales y ejecutar nuestras cargas de trabajo en un tiempo determinado

Azure Virtual network (vNet)â€‹

Las virtual networks en azure, son redes isoladas donde podemos lanzar nuestros servidores o servicios. Generalmente usan mascara 16

Subnetsâ€‹

Son un subgrupo isolado dentr de las virtual network, generalmente usan mascara 24. las subnets publicas son las que podemos conectarnos a internet las privadas son las que no tienen acceso a internet
